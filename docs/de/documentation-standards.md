# Dokumentationsstandard

EvoUI folgt dem dDocs package documentation standard.

## Lokalen

Release documentation nutzt `docs/en`, `docs/uk`, `docs/pl`, `docs/de` und
`docs/fr`. `docs/ua` wird nicht erstellt; legacy input `ua` wird zu `uk`
normalisiert.

## Seiten

Jede release locale enthaelt `README.md`, `user-guide.md`,
`developer-guide.md`, `frontend-guide.md`, `configuration.md`, `reference.md`,
`troubleshooting.md` und `documentation-standards.md`.

Deep implementation contracts koennen English-first in `docs/en` bleiben, aber
localized pages muessen darauf verlinken.
